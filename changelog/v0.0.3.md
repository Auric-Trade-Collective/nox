# Nox (Version 0.0.3)

## Added

**Environment Variables**
Nox v0.0.3 adds options to allow your API modules to access certain variables. The first
part of configuring variables is to list them in your nox.toml:

```toml
[nox.env]
test = { test = "Hi!" }
```

After you add this, you can access the new exposed functions:

```c
static inline char *RegisterName(NoxEndpointCollection *coll, char *name);

//usage
char *mySecret = RegisterName(coll, "test");
/*
    Keep your secret stored inside your API module. Furthermore, the "name" must
    match the key provided in the nox.env. In this case "test"
*/
```

Now your API can access variables set inside of the toml file:
```c
char *var = GetEnv(mySecret, "test"); //returns "Hi!"
```

**--debug**
Nox now has a ``--debug`` flag you can append as an argument. This will display
whatever debug information exists in the code/has been left by the developers.

Furthermore this function was exposed to the ABI:

```c
void LogDebug(char *name_space, char *msg);
```

**Redirect functions**
Redirect functions finally implemented.

```c
void TemporaryRedirect(HttpResponse *resp, HttpRequest *req, char *loc);
void PermanentRedirect(HttpResponse *resp, HttpRequest *req, char *loc);
```

## Patched

**Auth path issue**
The auth function previously didn't always load, depending on where you were
running nox from. Now it runs just fine and expands directories correctly.

**Logging Freeze Issue**
Up until now, logging was buggy and honestly not a priority inside of API
modules. Turns out it was simply an stdout flushing issue. Logging has been
updated to flush the contents of stdout so that messages actually appear.
